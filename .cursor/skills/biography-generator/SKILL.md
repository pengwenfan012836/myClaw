# Biography Generator Skill

根据人名，自动整理完整传记资料库。支持单人或批量处理。

---

## 核心原则

1. **演讲/访谈文件**：必须是完整的对话/演讲原文，不能是总结提炼
2. **传记空白时**：用第三人称叙述人生经历
3. **不要提炼**：不要生成"核心内容详解"、"经典语录"等总结性内容，直接展示原文
4. **按时间线分段**：以有完整原文的文件为分界线，合并中间无原文的年份
5. **多维度收集**：不仅收集演讲访谈，还收集童年、学生时代、亲友回忆等

---

## 人物照片与一句话描述

### 要求

1. **人物照片**：为每个已完成的传记人物添加一张人物照片
2. **照片存放位置**：`Biography/[人名]/images/` 目录下，命名为 `portrait.jpg`
3. **一句话描述**：在传记文件开头，用一句话概括这个人或代表观点

### 一句话描述模板

```markdown
# [人名] [英文名]

> "一句话描述或代表观点"
> —— [人名]

![人物照片](images/portrait.jpg)

---
```

### 一句话描述示例

- 💬 **Elon Musk** — "既然必须穿越地狱，那就继续前进。"
- 💬 **雷军** — "站在风口上，猪都能飞起来。"
- 💬 **马云** — "今天很残酷，明天更残酷，后天很美好。"
- 💬 **董明珠** — "格力空调世界第一，格力电器第二。"
- 💬 **任正非** — "我们除了胜利，已经无路可走。"

### 照片来源

**推荐网站**：

- 🖼️ **官方照片** — 维基百科、官方新闻发布会
- 📰 **新闻照片** — 新华社、Reuters、Getty Images
- 🎨 **高质量肖像** — 各公司年报、公开演讲截图

## 内容分类

所有有时间线的内容都放在"演讲与访谈"目录下：

### 1. 早年岁月回忆

- **童年经历**：如果有，自己讲述或他人（家人、邻居、老师）讲述
- **学生时代**：在学校期间的经历，与同学、老师的关系
- **命名格式**：`[起始年份]-[结束年份]-[一句话总结].md`

### 2. 亲友眼中的他

- 父母、兄弟姐妹、配偶、前配偶、合作伙伴、员工、同学等
- **命名格式**：`成长过程-[一句话总结].md`

### 3. 演讲与访谈

- **完整原文**：必须是完整的对话/演讲原文
- **合并叙述**：无完整原文的年份合并为"叙述"文件

---

## 分段合并规则

### 合并逻辑

以**有完整原文的文件**为分界线，将中间无原文的年份合并为一个"叙述"文件。

### 一句话总结命名要求

- 简洁明了，8-15字
- 体现该时期的核心主题或转折
- 避免使用"关键时刻"、"叙述"等通用词

**命名示例**：

- 📅 **2002-2008** — `2002-2008-梦想起源太空梦与能源梦.md`
- 📅 **2009-2012** — `2009-2012-绝处逢生商业航天黎明.md`
- 📅 **2014-2017** — `2014-2017-星辰大海可回收火箭时代.md`
- 📅 **2018-2025** — `2018-2025-风云变幻从首富到政商跨界.md`
- 👶 **童年** — `1971-1994-童年阴影被霸凌的少年.md`
- 🎓 **学生时代** — `1989-1995-学生时代从加拿大到斯坦福.md`
- 💭 **亲友回忆** — `成长过程-亲友回忆他们眼中的Musk.md`

## 信息收集策略

### 早年岁月信息

**信息来源**：

- 👶 **童年回忆** — 本人采访、父母/兄弟姐妹回忆录、邻居/老师采访
- 🎓 **学生时代** — 本人采访、同学/室友回忆、老师评价
- 💭 **亲友眼中的他** — 父母、配偶、前配偶、兄弟姐妹、合作伙伴、员工、同学、传记作者

**推荐信息源**：

- 🎤 **本人采访** — TED、Bloomberg、Recode、Lex Fridman Podcast
- 📚 **家人回忆录** — 亚马逊，当当
- 📖 **传记** — 维基百科、传记作家作品
- 👥 **员工/同事回忆** — Forbes、Business Insider、LinkedIn

---

### 演讲/访谈原文

**核心原则**：获取完整的原始文字记录，而非提炼总结

**推荐信息源**：

- 🎬 **TED/公开演讲** — TED.com 官方 transcripts、B站字幕
- 🏢 **官方发布会** — Apple/Google等官网档案、YouTube字幕
- 📺 **访谈节目** — B站、YouTube、自动字幕
- 📰 **杂志专访** — Playboy、Time 等 archive.org 原版
- 🎤 **新闻发布会** — 白宫/NASA等官方新闻发布会记录
- 💼 **股东大会** — 投资者关系页面、年报
- 🇨🇳 **中文资料** — B站视频字幕、知乎整理

---

### 传记文件信息收集

**信息来源**：Wikipedia、百度百科

**包含模块**：

1. 个人基本信息
2. 人生时间线
3. 核心成就
4. 核心特质
5. 管理风格（可选）
6. 推荐阅读
7. 相关链接

---

## 分步骤工作流

1. ✅ **Step 1** — 从 Wikipedia/维基百科提取 → 生成传记基础信息
2. ✅ **Step 2** — 添加人物照片和一句话描述 → 更新传记文件开头
3. ✅ **Step 3** — 搜索整理早年岁月 → 生成童年、学生时代、亲友回忆文件
4. ✅ **Step 4** — 搜索整理 → 生成时间线索引（区分有/无演讲访谈的年份）
5. ✅ **Step 5** — 查找演讲/访谈原文 → 生成完整原文内容
6. ✅ **Step 6** — 识别原文分界线 → 确定需要合并的区间
7. ✅ **Step 7** — 合并无原文区间 → 生成叙述文件
8. ✅ **Step 8** — 更新索引文件 → 按时间线排列所有文件

**信息深度**：详细深入型（演讲/访谈要求原文）

## 工作流程

### 单人处理

```
用户输入人名（如：黄峥）
    │
    ▼
1. 创建目录 Biography/[人名]/
    │
    ▼
2. 收集人物照片 → images/portrait.jpg
    │
    ▼
3. 从 Wikipedia/百度百科 提取信息
    │
    ▼
4. 生成传记文件（包含一句话描述和照片）
   └── [人名]传记.md
    │
    ▼
5. 创建演讲与访谈目录
   └── 演讲与访谈/
    │
    ▼
6. 收集早年岁月信息
   └── 童年、学生时代、亲友回忆
    │
    ▼
7. 整理所有有演讲/访谈的年份
   └── 生成完整原文内容
    │
    ▼
8. 以有原文的文件为分界线，合并中间无原文的年份
   └── 生成叙述文件
    │
    ▼
9. 更新索引文件
    │
    ▼
10. 全部完成后，汇总展示给用户
```

### 批量处理

```
用户提供人名表格或列表
    │
    ▼
1. 解析人名列表
    │
    ▼
2. 按顺序依次处理每个人名
   └── 重复单人处理流程
    │
    ▼
3. 每完成一个，展示进度
    │
    ▼
4. 全部完成后，汇总展示完整目录
```

## 使用说明

### 单人模式
1. 用户输入人名，直接开始自动执行
2. 无需用户确认，一路自动完成

### 批量模式
1. 用户提供人名表格或列表
2. 解析后按顺序依次处理每个人名
3. 每完成一个展示进度
4. 全部完成后汇总展示

## 注意事项

- 演讲/访谈必须是完整原文，禁止提炼总结
- 无演讲/访谈的年份用第三人称叙述人生经历
- 以有完整原文的文件为分界线，合并中间无原文的年份
- 叙述文件名格式：`[起始年份]-[结束年份]-[一句话总结].md`
- 一句话总结要简洁（8-15字），体现核心主题
- 第三人称叙述要扩展和发散，包含心理描写、细节、对话等
- 早年岁月文件要注明信息来源（本人/家人/朋友/同事等）
- **所有有时间线的内容都放在"演讲与访谈"目录下**
- 所有内容必须基于可靠来源
- 涉及日期务必核实准确性
- 语录引用需注明来源
- 视频/文章链接需可用
- 文件编码统一使用 UTF-8
- **必须为每个人物添加照片**：`images/portrait.jpg`
- **必须在传记开头添加一句话描述**
