# Biography Generator Skill

根据人名，自动整理完整传记资料库。支持单人或批量处理。

---

## 核心原则

1. **演讲/访谈文件**：必须是完整的对话/演讲原文，不能是总结提炼
2. **传记空白时**：用第三人称叙述人生经历
3. **不要提炼**：不要生成"核心内容详解"、"经典语录"等总结性内容，直接展示原文
4. **按时间线分段**：以有完整原文的文件为分界线，合并中间无原文的年份
5. **多维度收集**：不仅收集演讲访谈，还收集童年、学生时代、亲友回忆等

---

## 目录结构

```
Biography/[人名]/
├── [人名]传记.md
├── [人名]演讲访谈索引.md
└── 演讲与访谈/
    ├── [起始年份]-[结束年份]-[一句话总结].md   # 合并叙述文件
    ├── [年份]-[事件名].md                      # 有完整原文的文件
    ├── [年份]-[事件名].md
    ├── [年份]-[事件名].md                      # 童年/学生时代回忆
    └── 成长过程-[一句话总结].md                # 亲友回忆
```

**注意**：所有有时间线的内容都放在"演讲与访谈"目录下，不再单独创建"早年岁月"文件夹。

---

## 内容分类

所有有时间线的内容都放在"演讲与访谈"目录下：

### 1. 早年岁月回忆

- **童年经历**：如果有，自己讲述或他人（家人、邻居、老师）讲述
- **学生时代**：在学校期间的经历，与同学、老师的关系
- **命名格式**：`[起始年份]-[结束年份]-[一句话总结].md`

### 2. 亲友眼中的他

- 父母、兄弟姐妹、配偶、前配偶、合作伙伴、员工、同学等
- **命名格式**：`成长过程-[一句话总结].md`

### 3. 演讲与访谈

- **完整原文**：必须是完整的对话/演讲原文
- **合并叙述**：无完整原文的年份合并为"叙述"文件

---

## 分段合并规则

### 合并逻辑

以**有完整原文的文件**为分界线，将中间无原文的年份合并为一个"叙述"文件。

### 一句话总结命名要求

- 简洁明了，8-15字
- 体现该时期的核心主题或转折
- 避免使用"关键时刻"、"叙述"等通用词

**命名示例**：

| 包含年份 | 合并文件名 |
|----------|------------|
| 2002-2008 | `2002-2008-梦想起源太空梦与能源梦.md` |
| 2009-2012 | `2009-2012-绝处逢生商业航天黎明.md` |
| 2014-2017 | `2014-2017-星辰大海可回收火箭时代.md` |
| 2018-2025 | `2018-2025-风云变幻从首富到政商跨界.md` |
| 童年 | `1971-1994-童年阴影被霸凌的少年.md` |
| 学生时代 | `1989-1995-学生时代从加拿大到斯坦福.md` |
| 亲友回忆 | `成长过程-亲友回忆他们眼中的Musk.md` |

## 信息收集策略

### 早年岁月信息

**信息来源**：

| 类型 | 来源 |
|------|------|
| 童年回忆 | 本人采访、父母/兄弟姐妹回忆录、邻居/老师采访 |
| 学生时代 | 本人采访、同学/室友回忆、老师评价 |
| 亲友眼中的他 | 父母、配偶、前配偶、兄弟姐妹、合作伙伴、员工、同学、传记作者 |

**推荐信息源**：

| 类型 | 推荐网站/书籍 |
|------|--------------|
| 本人采访 | TED、Bloomberg、Recode、Lex Fridman Podcast |
| 家人回忆录 | 亚马逊，当当 |
| 传记 | 维基百科、传记作家作品 |
| 员工/同事回忆 | Forbes、Business Insider、LinkedIn |

---

### 演讲/访谈原文

**核心原则**：获取完整的原始文字记录，而非提炼总结

**推荐信息源**：

| 类型 | 推荐网站 |
|------|----------|
| TED/公开演讲 | TED.com 官方 transcripts、B站字幕 |
| 官方发布会 | Apple/Google等官网档案、YouTube字幕 |
| 访谈节目 | B站、YouTube、自动字幕 |
| 杂志专访 | Playboy、Time 等 archive.org 原版 |
| 新闻发布会 | 白宫/NASA等官方新闻发布会记录 |
| 股东大会 | 投资者关系页面、年报 |
| 中文资料 | B站视频字幕、知乎整理 |

## 分步骤工作流

| 步骤 | 操作 | 目标 |
|------|------|------|
| Step 1 | 从 Wikipedia/维基百科提取 | 生成传记基础信息 |
| Step 2 | 搜索整理早年岁月 | 生成童年、学生时代、亲友回忆文件 |
| Step 3 | 搜索整理 | 生成时间线索引（区分有/无演讲访谈的年份） |
| Step 4 | 查找演讲/访谈原文 | 生成完整原文内容 |
| Step 5 | 识别原文分界线 | 确定需要合并的区间 |
| Step 6 | 合并无原文区间 | 生成叙述文件 |
| Step 7 | 更新索引文件 | 按时间线排列所有文件 |

**信息深度**：详细深入型（演讲/访谈要求原文）

## 工作流程

### 单人处理

```
用户输入人名（如：张一鸣）
    │
    ▼
1. 创建目录 Biography/[人名]/
    │
    ▼
2. 生成传记文件
   └── [人名]传记.md
    │
    ▼
3. 创建演讲与访谈目录
   └── 演讲与访谈/
    │
    ▼
4. 收集早年岁月信息
   └── 童年、学生时代、亲友回忆
    │
    ▼
5. 整理所有有演讲/访谈的年份
   └── 生成完整原文内容
    │
    ▼
6. 以有原文的文件为分界线，合并中间无原文的年份
   └── 生成叙述文件
    │
    ▼
7. 更新索引文件
    │
    ▼
8. 全部完成后，汇总展示给用户
```

### 批量处理

```
用户提供人名表格或列表
    │
    ▼
1. 解析人名列表
    │
    ▼
2. 按顺序依次处理每个人名
   └── 重复单人处理流程
    │
    ▼
3. 每完成一个，展示进度
    │
    ▼
4. 全部完成后，汇总展示完整目录
```

## 使用说明

### 单人模式
1. 用户输入人名，直接开始自动执行
2. 无需用户确认，一路自动完成

### 批量模式
1. 用户提供人名表格或列表
2. 解析后按顺序依次处理每个人名
3. 每完成一个展示进度
4. 全部完成后汇总展示

---

## 注意事项

- 演讲/访谈必须是完整原文，禁止提炼总结
- 无演讲/访谈的年份用第三人称叙述人生经历
- 以有完整原文的文件为分界线，合并中间无原文的年份
- 叙述文件名格式：`[起始年份]-[结束年份]-[一句话总结].md`
- 一句话总结要简洁（8-15字），体现核心主题
- 第三人称叙述要扩展和发散，包含心理描写、细节、对话等
- 早年岁月文件要注明信息来源（本人/家人/朋友/同事等）
- **所有有时间线的内容都放在"演讲与访谈"目录下**
- 所有内容必须基于可靠来源
- 涉及日期务必核实准确性
- 语录引用需注明来源
- 视频/文章链接需可用
- 文件编码统一使用 UTF-8
